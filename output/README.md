# Output folder — what's in here

This folder holds the finished deliverables the engine produces each cycle.
If you are reviewing this project, start with the report.

## The report (start here)

**`Credit_Risk_Report_<period>`** — the same report in three formats:

| File | Open it in | Best for |
| --- | --- | --- |
| `Credit_Risk_Report_Q2_2026.md` | GitHub / any text viewer | Quick read in the browser |
| `Credit_Risk_Report_Q2_2026.html` | Any web browser | Formatted, self-contained |
| `Credit_Risk_Report_Q2_2026.docx` | Microsoft Word | The version a committee reads |

The report opens with a plain-English executive summary and a "how to read
the numbers" guide, then six sections of credit-risk benchmarks for
Australian lenders:

1. **PD** — probability a borrower defaults within 12 months, by segment.
2. **LGD** — share of the loan not recovered after a default.
3. **Expected loss** — the headline loss rate per segment (`PD × LGD`).
4. **Stress testing** — PD and LGD under a downturn scenario.
5. **Portfolio monitoring** — arrears, non-performing, impaired and loss rates.
6. **Per-bank industry exposures** — Big 4 lending and bad debts by industry.

Every number traces back to a named public disclosure (bank Pillar 3
reports, APRA, the RBA, S&P) and a reporting date — nothing is estimated
or adjusted.

## The data (`data/`)

The same numbers as machine-readable CSVs, ready to load into a pricing or
provisioning model:

| File | Contents |
| --- | --- |
| `pd_inputs.csv` | Probability-of-default observations, one row per source |
| `lgd_inputs.csv` | Loss-given-default observations, one row per source |
| `expected_loss_inputs.csv` | Median PD, median LGD and expected-loss rate per segment |
| `stress_testing_inputs.csv` | Base and stressed PD / LGD / expected-loss rates |
| `portfolio_monitor_inputs.csv` | Arrears, NPL, impaired and loss-rate metrics |
